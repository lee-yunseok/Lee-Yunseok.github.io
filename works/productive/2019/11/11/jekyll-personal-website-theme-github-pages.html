지킬 개인 웹사이트 테마(Jekyll Personal Website Theme)

이 테마는 깃허브 공식 personal website를 지킬(Jekyll)을 중점으로 사용해 보다 전문적인 블로거에 친화적으로 기능하도록 확장한 포트입니다. 카드형 아카이브와 최근 글을 지원합니다. 각각의 포스트에는 이미지 슬라이더와 이전 및 다음 글, 관련 글, 댓글을 지원합니다. 카드 아카이브는 카테고리별 페이지네이션을 사용할 수 있습니다. 쉽게 메타데이터와 애널리틱스, 소유권 확인, RSS 피드와 사이트맵을 사용하도록 설계됐습니다. 컨택트 폼 또한 지원합니다.


당신의 소프트웨어 개발 기술을 시연

이 레포지토리는 당신이 소프트웨어 개발자로서 시연할 수 있는 개인 웹사이트를 빠르게 시작할 수 있는 코드를 제공합니다. 또한 당신이 깃허브 레포지토리의 코드를 관리할 때, 당신의 아바타와 레포지토리들은 자동으로 프로필에 렌더될 것입니다.

당신의 개인 웹사이트는 개인화되길 기다립니다. 언어 또는 산업과 같은 소프트웨어 개발에 대한 특정 관심 영역을 강조할 수 있는 공간이 포함되어 있습니다. 그리고 그것은 당신의 멋진 블로그 게시물을 게시할 준비가 되었습니다.

이러한 것들은 Jekyll (웹사이트 구축을 위해), GitHub Pages (웹사이트 호스팅을 위해), 그리고 GitHub’s API (자동으로 당신의 웹사이트와 콘텐츠를 채우기 위해) 조합으로 가능합니다.


설치

lee-yunseok/jekyll-personal-website 레포지토리에 포크를

당신의 “개인 웹사이트 스타터” 레포지토리 복사를 만드는 것으로 당신의 프로젝트를 사용자 정의할 수 있습니다. “fork”는 레포지토리를 복사합니다. 그렇기 때문에 github/personal-website 레포지토리에 “Fork”를 찍으십시오.

한 번 당신이 포크된 레포지토리를 찾으면 그것은 당신의 것입니다. 당신이 주인이기에, 당신이 원하면 그것을 공개할 수 있습니다.


로컬 개발 환경 설치

당신의 웹사이트를 로컬 웹 개발 환경에서 관리하길 원한다면 Ruby를 사용해야 합니다.

포크된 레포지토리를 찾았다면 그것을 복제하십시오.


Jekyll 설치

Jekyll은 대부분의 시스템에서 Ruby Gem으로 설치됩니다.

  1. Ruby 전체 개발 환경 설치
  2. Jekyll과 bundler gems 설치
    gem install jekyll bundler
  3. 새로운 디렉토리로 변경
    cd jekyll-personal-website
  4. 누락된 gem 설치
    bundle install
  5. 로컬 서버에서 사용 가능한 사이트 빌드
    bundle exec jekyll serve

    아래와 비슷한 것을 보게 됩니다:

    Configuration file: /octocat/personal-website/_config.yml
         Source: /octocat/personal-website
    Destination: /octocat/_site

    “No GitHub API authentication could be found” 메시지를 걱정하지 마십시오. 브랜치 이름과 같은 더 자세한 메타데이터를 표시하려는 경우 참고하세요. API authentication is only necessary

  6. 이제 로컬 웹에 접속하십시오. http://localhost:4000


공개

당신이 당신의 웹사이트 코드를 깃허브에 호스트할 때, 당신은 깃허브 페이지를 통해 무료 호스팅을 지원받을 수 있습니다.

가장 빠른 방법은 당신의 레포지토리 이름을 username.github.io로 변경하는 것으로, username은 당신의 깃허브 유저 이름(혹은 단체 이름)입니다. 그러면, 다음에 어떤 변경 사항을 당신의 레포지토리 master 브랜치에 push할 때, 그것들은 웹상에서 당신의 username.github.io 주소로 접속할 수 있습니다.

커스텀 도메인을 사용하려면, 당신의 깃허브 레포지토리의 “Custom domain” 설정에 추가하십시오. 그러면 등록이 됩니다/되거나 도메인과 DNS 공급자 설정을 해야 합니다.


사용자 정의

당신의 웹사이트입니다. 당신이 소스 코드를 제어할 수 있습니다. 그러므로 당신이 원한다면 모든 것을 사용자 정의할 수 있습니다. 하지만 우리는 당신의 웹사이트를 세상에 공개할 빠른 사용자 정의를 제공합니다.


빠른 설정 변경

대부분의 사용자 정의는 당신의 레포지토리의 _config.yml 파일을 수정하는 몇 초의 작업만이 필요합니다. 다만 Jekyll 기반 웹 사이트가 올바르게 다시 빌드되도록 새 변경 사항을 저장할 때마다 로컬 서버를 다시 시작하십시오.

  1. 키보드로 CTRL+c 커맨드를 입력해 서버를 내립니다.
  2. 서버를 재시작하는 커맨드: bundle exec jekyll serve


레이아웃

웹 사이트는 기본적으로 큰 화면 장치에서 사진, 이름 및 기본 정보가 왼쪽 정렬 된 “사이드 바”로 표시되는 2열 레이아웃으로 표시됩니다. _config.yml 파일에서 layout : sidebar 를 찾아 layout : stacked로 변경하여 “스택 된” 단일 열 레이아웃으로 빠르게 전환할 수 있습니다.


스타일

웹 사이트는 기본적으로 “light” 흰색과 회색 배경, 어두운 텍스트로 보여집니다. _config.yml 파일에서 style : light를 style : dark로 변경하면 흰색 텍스트가 있는 “어두운” 배경으로 빠르게 전환할 수 있습니다.


타임존

타임존 을 아래처럼 작성하면 당신의 포스트들의 작성 시간이 당신의 시간에 맞게 표시됩니다.

timezone: Asia/Seoul


깃허브 사용자 설정

이 테마는 당신의 깃허브 유저 정보를 입력해야 합니다.

repository: 깃허브_로그인_이름/당신의_레포지토리
name: 당신의 이름
email: 당신의 전자우편 주소. 웹사이트에 전자우편 주소가 보이길 원하지 않는다면 비워두세요.
bio: 당신의 간단한 소개.
org: 당신의 깃허브 단체. 단체가 없다면 비워두세요.
location: 당신의 거주지역.
hireable: "true" 또는 "false"로 설정해 일을 원하는지 표시할 수 있습니다.


기본 메타데이터 태그

아래의 것들은 <head> </head>를 채웁니다.

title: 당신의 웹사이트 제목
description: 당신의 웹사이트 설명
keyword: 당신의 웹사이트 키워드


프로젝트

“My Projects” 섹션은 당신이 “push”한 레포지토리의 순서대로 당신의 웹사이트에 기본적으로 생성됩니다. 그것은 기본적으로 당신이 포크한 것을 제외합니다. 하지만 이러한 파라미터들은 _config.yml파일의 projects 라인에서 빠르게 설정할 수 있습니다.

파라미터가 포함하는 것:

  • sort_by: 레포지토리가 정렬되는 순서입니다. pushed와 stars 옵션이 있습니다.
  • limit: 웹사이트의 “My Projects” 섹션에 보여질 레포지토리의 수를 제어합니다. 이 숫자는 9로 기본 설정돼 있습니다.
  • exclude:
    • forks: 값이 true일 때, 포크한 레포지토리는 리스팅에서 제외됩니다.
    • projects: 리스팅에 제외되길 바라는 레포지토리의 이름을 적습니다.


토픽

당신의 웹사이트는 “My Interests.” 섹션에 보여질 세 개의 토픽이 사전 설정돼 있습니다. (“Web design”과 “Sass”) 이것들은 또한 _config.yml파일에 정의되며, 당신의 토픽의 이름과 함께 아래의 값을 정의할 수 있습니다:

  • web_url: 토픽에 링크되길 원하는 웹 주소입니다. (https://github.com/topics/sass).
  • image_url: 토픽에 보여질 이미지의 웹 주소입니다.


소셜 미디어

당신의 웹사이트는 당신이 사용하는 소셜 미디어 서비스의 링크를 지원합니다. Behance, Dribbble, Facebook, LinkedIn, Medium, Stack Overflow, Twitter, 그리고 YouTube. 당신이 사용하는 서비스를 설정하세요:

  1. 레포지토리의 _config.yml 파일을 엽니다.
  2. 사전 구성된 social_media 라인을 key: value 값으로 편집합니다:
social_media:
  behance: 당신의_유저네임
  dribbble: 당신의_유저네임  
  facebook: 당신의_유저네임
  hackerrank: 당신의_유저네임
  instagram: 당신의_유저네임
  keybase: 당신의_유저네임
  linkedin: 당신의_유저네임
  medium: 당신의_유저네임
  stackoverflow: 당신의_유저ID
  telegram: 당신의_유저네임
  twitter: 당신의_유저네임
  unsplash: 당신의_유저네임
  vk: 당신의_유저네임
  website: 당신의_웹사이트_주소
  youtube: 당신의_유저네임

정의한 각 서비스에 대한 프로필 링크는 웹 사이트의 <header>에 표시되며 바이오에 추가됩니다. 해당 서비스가 공유를 지원하는 경우 게시하는 모든 블로그 게시물에는 각 소셜 미디어 서비스를 사용하여 해당 게시물을 공유하는 링크가 포함됩니다.

참고: 이 기능은 당신의 레포지토리의 두 개의 파일에 의해 지원됩니다:

  • /_data/social_media.yml: 변수 이름, 표시 이름, URL 경로 및 SVG 아이콘을 포함하여 지원되는 각 서비스를 정의합니다.
  • /_includes/social_media_share_url.html: 공유 URL을 지원하는 소셜 미디어 서비스에 필요한 공유 URL을 출력합니다.

이 리포지토리에서 아직 지원되지 않는 소셜 미디어 서비스를 추가하려는 경우이 두 파일을 편집하여 해당 지원을 구축할 수 있습니다.


페이스북 어드민

이것은 누군가 당신의 포스트를 공유할 때 당신의 페이스북 페이지 메타데이터 를 사용합니다. 원하지 않는다면 비워두세요.

facebook_admins: 당신의_페이스북_어드민_넘버


디스커스(Disqus) 댓글

사람들은 당신의 포스트에 댓글을 남길 수 있습니다. 당신의 사이트를 추가하고 디스커스 숏네임만 적으면 됩니다. 원하지 않는다면 비워두세요.

disqus: 당신의_디스커스_숏네임

웹사이트 소유권 확인

소유권 확인 코드만 넣으면 작동합니다.

google_verification: 소유권_확인_코드
bing_verification: 소유권_확인_코드
naver_verification: 소유권_확인_코드
norton_verification: 소유권_확인_코드


구글 애널리틱스

간단하게 추적 코드를 넣으세요.

google_analytics: 추적_코드


About 페이지

_/includes/about.html 파일을 HTML과 Liquid를 사용해 편집하여 당신에 대해 보다 많은 정보를 포함할 수 있습니다.


페이지 추가

당신의 웹사이트에 페이지를 추가하는 방법 (예: 견적문의):

  1. 레포지토리의 /_pages/ 안에 .html 또는 .md파일을 만듭니다.
  2. 파일의 이름은 당신이 원하는 페이지의 URL로 사용될 것으로 정하세요. (예: http://yourname.github.io/pages/파일이름).
  3. 파일의 시작에는 다음과 같은 서문를 사용합니다:
  ---
  layout: pages
  title: "페이지 제목"
  description: "페이지 설명"
  permalink: pages/페이지_이름/
  ---
  1. Liquid를 사용하길 원한다면 당신의 페이지 파일에는 서문만 적으시고, _includes 안에 파일을 만들어 콘텐츠를 작성하세요. 그리고 페이지 파일로 돌아와 서문에 include_html: 당신의_LIQUID_파일를 추가하세요.


카테고리 페이지 추가

당신의 웹사이트에 블로그 메뉴와 같은 카테고리를 추가할 수 있습니다:

  1. 레포지토리의 /_categories/ 안에 다음과 같은 서문을 포함하는 새 .html 또는 .md 파일을 만드십시오.
  ---
  layout: categories
  title: "카테고리 제목"
  description: "카테고리 설명"
  permalink: 카테고리 이름
  pagination:
    enabled: true
    category: 카테고리 이름
  ---
  1. _config.yml에 카테고리 링크와 옥티콘을 추가하세요.
  - title: 카테고리 이름
    octicon: 옥티콘 이름
    url: 카테고리 이름


블로그 포스트 작성

웹사이트에 블로그 포스트를 추가하는 방법:

  1. 레포지토리의 /_posts/ 디렉토리 안에 새로운 .md 파일을 만듭니다.
  2. 파일의 이름은 다음의 포맷을 따릅니다:
년-월-일-제목.md
  1. 파일의 시작에는 다음과 같은 서문을 작성하십시오:
---
layout: post
title: "포스트 제목"
date: 년-월-일 시:분:초
description: '포스트의 메타 설명(부제 또는 간력한 내용 등)'
og_type: article
tags:
- 태그
categories:
- 카테고리
twitter_text: '트위터 카드에 보일 설명'
introduction: 'Thought 섹션 또는 카테고리 페이지에 보일 중략 내용'
---

웹 사이트에는 참조할 수 있는 기본 블로그 게시물이 있습니다. 특히, 그것의 서문은 published를 false로 선언하여 당신의 웹사이트에 나타나지 않도록 합니다.

서문에서 layout을 정의할 수 있지만, 웹사이트는 / _posts / 디렉토리의 모든 게시물에 post 레이아웃을 할당하도록 미리 구성되어 있습니다. 따라서 게시물에서 이를 선언하지 않아도 됩니다.

블로그 게시물 작성 및 관리에 대한 Jekyll의 규칙은 매우 유연합니다. Jekyll의 “게시물”에 대한 자세한 내용을 볼 수 있습니다.

Thought 섹션이나 카테고리 페이지에서 썸네일로 사용할 이미지나 비디오를 지정할 수도 있습니다. image : YOUR_IMAGE_URL 또는 youtube : YOUTUBE_VIDEO_URL, vimeo : VIMEO_VIDEO_URL, video : YOUR_VIDEO_URL을 서문에 추가하세요.


포스트에 jQuery lightSlider 사용

테마가 제공하는 jQuery lightSlider로 포스트에 이미지 슬라이더를 적용합니다. 포스트의 원하는 곳에 다음과 같은 코드를 작성하세요.

<div class="yslider">
  <ul>
    <li>
      <img src="이미지의 웹주소" alt="이미지에 대한 설명" />
    </li>
    ## 위처럼 <li><img></li>를 순서대로 추가 ##
  </ul>
</div>

Formspree 컨택트 폼 사용

/_pages/contact.md을 열고 YOUR_EMAIL_ADDRESS를 당신의 전자우편 주소로 변경하세요. 이것으로 사람들이 Formspree를 통해 당신에게 문의 우편을 보낼 수 있습니다.


개인정보 처리방침

개인정보 처리방침을 /_pages/privacy-policy.md 파일에 작성하십시오. Privacy Policy Generator를 사용해 간단히 만들 수 있습니다.


스타일

당신의 웹사이트는 GitHub’s very flexible CSS framework called “Primer,”를 사용해 styles.scss 파일 안에 사전 구성돼 있습니다. 다음 규칙으로 다른 CSS를 포함할 수 있습니다:

@import url('https://unpkg.com/primer/build/build.css');

물론 당신은 이것을 제거하거나 다른 프레임워크로 교체할 수 있습니다. 귀하의 웹 사이트에 미리 제공된 HTML은 열 너비, 여백 및 배경색과 같은 것을 정의하기 위해 여러 프라이머 “유틸리티 클래스”를 참조합니다.

/assets/styles.scss Sass 스타일 시트에 커스텀 CSS를 추가하여 프라이머의 스타일을 추가하고 확장하는 옵션도 있습니다. 이 파일을 편집하면 웹사이트의 색 구성표, 타이포그래피 등을 사용자 정의할 수 있습니다.
